# المرحلة ١: أنشئ "الدفتر" (Firebase) — ١٥ دقيقة

الدفتر هو المكان اللي يخزّن أرقام محافظك على الإنترنت.

1. افتح المتصفح → روح **console.firebase.google.com**
2. سجّل دخول بحساب Google حقك.
3. اضغط **Add project** (أضف مشروع).
4. اسم المشروع: اكتب `mahafez` → **Continue**.
5. صفحة Google Analytics → اضغط الزر عشان **تعطّله** (Disable) → **Create project**.
6. انتظر دقيقة → **Continue**.

✅ صار عندك دفتر فاضي.

---

# المرحلة ٢: فعّل قاعدة البيانات — ٥ دقايق

1. من القائمة اليسار → **Build → Firestore Database**.
2. اضغط **Create database**.
3. اختر **Production mode** → **Next**.
4. الموقع (Location): اختر `eur3` → **Enable**.

✅ قاعدة البيانات جاهزة.

---

# المرحلة ٣: فعّل تسجيل الدخول — ٥ دقايق

1. من القائمة اليسار → **Build → Authentication**.
2. اضغط **Get started**.
3. اختر **Email/Password** → فعّل أول خيار (Enable) → **Save**.
4. فوق، اضغط تبويب **Users** → **Add user**.
   - Email: اكتب `me@mahafez.local`
   - Password: اختر كلمة مرور **واكتبها في مكان آمن** (بتستخدمها كثير).
   - **Add user**.

✅ صار عندك مستخدم تدخل فيه.

---

# المرحلة ٤: انسخ إعدادات مشروعك — ٥ دقايق

1. فوق يسار، اضغط **⚙️ (الترس) → Project settings**.
2. انزل تحت لـ **Your apps** → اضغط أيقونة **`</>`** (الويب).
3. App nickname: اكتب `mahafez-web` → **Register app**.
4. بيطلع لك كود فيه `firebaseConfig = { ... }` — **انسخه كامل**.
5. افتح ملف **`public/firebase-config.js`** (من الملفات اللي عطيتك).
6. الصق القيم مكان القيم الوهمية. لازم يصير شكله:
   ```
   export const firebaseConfig = {
     apiKey: "AIza....(قيمتك)",
     authDomain: "mahafez-xxxxx.firebaseapp.com",
     projectId: "mahafez-xxxxx",
     ...
   };
   export const AUTH_EMAIL = "me@mahafez.local";
   ```
7. احفظ الملف.

✅ التطبيق صار يعرف دفترك.

# المرحلة ٥: انشر التطبيق على الإنترنت — ١٥ دقيقة

هنا تحتاج "Terminal" (سطر الأوامر). لو ما تعرفه، لا تخاف — بس انسخ والصق.

**على الكمبيوتر:**

1. ثبّت أداة Firebase (مرة وحدة). افتح Terminal واكتب:
   ```
   npm install -g firebase-tools
   ```
   (لو ما عندك npm، ثبّت Node.js أول من nodejs.org)

2. سجّل دخول:
   ```
   firebase login
   ```
   (بيفتح المتصفح — سجّل بنفس حساب Google)

3. روح لمجلد المشروع:
   ```
   cd المسار/إلى/mahafez
   ```

4. اربط المشروع:
   ```
   firebase use --add
   ```
   (اختر `mahafez` من القائمة، وأعطه اسم مستعار `default`)

5. انشر:
   ```
   firebase deploy
   ```

6. بينتهي ويعطيك رابط مثل:
   ```
   https://mahafez-xxxxx.web.app
   ```
   **احفظ هذا الرابط** — هذا تطبيقك!

✅ التطبيق صار على الإنترنت.

> **بديل أسهل بدون Terminal:** لو الأوامر صعبة عليك، استخدم **Claude Code** —
> يسوي هالخطوات نيابة عنك خطوة بخطوة.

# المرحلة ٦: جهّز المحافظ — دقيقتين

1. افتح بالمتصفح: `https://mahafez-xxxxx.web.app/seed.html`
   (نفس رابطك + `/seed.html`)
2. اكتب كلمة المرور.
3. اضغط **أنشئ المحافظ + ذاكرة المتاجر**.
4. بتشوف رسائل: ✓ محفظة السكن، ✓ المقاضي... إلخ.

✅ الـ14 محفظة + 54 متجر جاهزين.

# المرحلة ٧: افتح التطبيق وجرّبه — دقيقتين

1. افتح: `https://mahafez-xxxxx.web.app`
2. اكتب كلمة المرور → **دخول**.
3. لازم تشوف محافظك وأرصدتها!
4. **أضفه للشاشة الرئيسية:** في سفاري، اضغط زر المشاركة (المربع بالسهم) →
   **Add to Home Screen**. يصير كأنه تطبيق حقيقي.

✅ تطبيقك شغّال!

# المرحلة ٨: اختبر الاستقبال — ٥ دقايق

قبل الشورت كت، تأكد إن الأنبوب شغّال:

1. افتح: `https://mahafez-xxxxx.web.app/test-ingest.html`
2. اكتب كلمة المرور.
3. اضغط **عيّنة: نقاط بيع** (يعبّي رسالة تجريبية).
4. اضغط **استخرج وأرسل لـ Firestore**.
5. لازم يطلع ✓ نجح.
6. ارجع للتطبيق الرئيسي — بتلقى العملية دخلت!

✅ لو نجح هذا، الشورت كت بينجح أكيد.

هذي أطول خطوة. اتبع ملف **`docs/SHORTCUT.md`** بالتفصيل، ومعك
**`docs/SNIPPETS.md`** للنسخ واللصق (كل الروابط والـregex جاهزة).

باختصار، بتسوي شيئين:
1. **شورت كت** اسمه "سجّل مصروف" — يستخرج المبلغ والمتجر ويرسلهم.
2. **أتمتة** — أي رسالة من `alinma` فيها "شراء" → تشغّل الشورت كت تلقائياً.

✅ من الحين، أي شراء يدخل التطبيق لحاله.

## لو علقت في أي خطوة

المرحلة ٥ (النشر) والمرحلة ٩ (الشورت كت) هما الأصعب. لو علقت فيهم:
- **استخدم Claude Code** — يشتغل على جهازك ويساعدك خطوة بخطوة عملياً.
- أو ارجع لي وقول لي وقفت عند أي مرحلة بالضبط، وأشرحها أكثر.

**تذكير:** التطبيق يسجّل ويصنّف. تحويل الفلوس الفعلي بين محافظ البنك يبقى بيدك
(أمان البنك يمنع الأتمتة). أداة تخطيط شخصية، مو استشارة مالية.
